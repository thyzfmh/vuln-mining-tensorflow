## Core Rules

1. Black-box only — no version hints, no bug hints, no CVE references in prompts to the LLM
2. Evidence-based — every vulnerability must have source code evidence and a reproducible trigger
3. AI-generated tests — any proof-of-concept or test must be generated by the AI, not hand-written
4. Complete interaction log — every LLM turn must be recorded in `llm_chat_log.json` without editing
5. Engineering methodology — systematic scanning > random poking; combining traditional SAST/DAST with LLM is a plus
6. Multi-layer scanning — cover input validation, integer overflow, null deref, resource exhaustion, type confusion, use-after-free
7. Real bugs only — no false positives; each bug must be provably triggerable
8. Autonomous mode — run full scan without asking questions until `final_verify.sh` passes
