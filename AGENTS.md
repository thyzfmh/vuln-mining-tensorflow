# AI Vulnerability Mining — TensorFlow

## Your Task

**Use AI to discover real vulnerabilities in TensorFlow.** Execute the skill at `work/skills/vuln_mining_tf_blackbox/` and follow it to completion.

- **Target**: `code/tensorflow` (C++)
- **Deliverables**: `vulnerability_list.md`, `llm_chat_log.json`, `vulnerability_report.md`, `verify/run_test.py`
- **Output directory**: `./`
- **Do not stop** until `final_verify.sh` passes
- **Do not ask the user questions** — make reasonable decisions autonomously
- **Black-box standard**: never tell the AI which bugs exist, which version this is, or hint at known vulnerabilities

## Entry Point

1. Read and follow `INSTRUCTION.md`
2. Load and execute `work/skills/vuln_mining_tf_blackbox/prompt.md`
3. Follow the 5-step pipeline: Semantic Analysis → Hypothesis → Test Generation → Runtime Verification → Evidence Collection
4. Keep going until `final_verify.sh` passes

## Core Rules

1. Black-box only — no version hints, no bug hints, no CVE references in prompts to the LLM
2. Evidence-based — every vulnerability must have source code evidence and a reproducible trigger
3. AI-generated tests — any proof-of-concept or test must be generated by the AI, not hand-written
4. Runtime verification — all vulnerabilities must be verified by running `verify/run_test.py`
5. Complete interaction log — every LLM turn must be recorded in `llm_chat_log.json` without editing
6. Engineering methodology — systematic scanning > random poking; combining traditional SAST/DAST with LLM is a plus
7. Multi-layer scanning — cover input validation, integer overflow, null deref, resource exhaustion, type confusion, use-after-free
8. Real bugs only — no false positives; each bug must be provably triggerable
9. Autonomous mode — run full scan without asking questions until `final_verify.sh` passes
