# AI Vulnerability Mining Competition Package

This repository is packaged for one task: use OpenCode to discover real,
runtime-verified vulnerabilities in the validation source tree under `code/`.

## Entry

Use the repository root entry file:

```text
INSTRUCTION.md
```

It tells OpenCode to load and execute:

```text
work/skills/vuln-mining-autonomous/SKILL.md
```

## Fixed Task

- Target source: auto-detected from `code/`
- Skill name: `vuln-mining-autonomous`
- Final gate: `python3 work/skills/vuln-mining-autonomous/scripts/final_verify.py`
- Required deliverables: `vulnerability_list.md`, `llm_chat_log.json`, `vulnerability_report.md`, `verify/run_test.py`

The skill owns the full loop: target discovery, inventory, deterministic SAST
candidates, black-box LLM review logging, AI-generated tests, runtime
verification, final reports, and result update.

## Directory Layout

```text
project-root/
├── INSTRUCTION.md
├── AGENTS.md
├── code/
│   └── .gitkeep
├── work/
│   └── skills/
│       └── vuln-mining-autonomous/
│           ├── SKILL.md
│           ├── scripts/
│           ├── templates/
│           └── tests/
├── result/
│   └── output.md
├── logs/
└── docs/
```

Runtime deliverables and reports are generated by the skill and are ignored in
git until a competition run produces final evidence.
